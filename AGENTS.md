# AGENTS.md

Guidelines for agentic coding assistants working in this repository.

## What this repo is

A Go multi-module workspace (`go.work`) containing focused, independently
versioned library modules: `apimanager`, `config`, `dependency`, `env`,
`executors`, `lists`, `metrics`, and `rest`. There is no single binary
or application — only importable packages.

## Repository structure

```text
go.work              → Workspace linking all modules
Makefile             → Developer tasks (tidy, lint, tag)
.golangci.yaml       → Root golangci-lint config (inherited by modules)
.pre-commit-config.yaml → Pre-commit hooks (format, vet, lint, test)
<module>/
  go.mod             → Module declaration and dependencies
  *.go               → Production code
  *_test.go          → Tests (same package for white-box, _test suffix for black-box)
  *_moq.go           → Generated mocks via moq — DO NOT EDIT by hand
  README.md          → Module-level documentation
example/             → Runnable examples, one sub-directory per module
```

## Build and tooling commands

### Tidy all modules

```bash
make tidy
```

Runs `go work sync` then `go mod tidy` in every module directory.

### Lint (all modules)

```bash
make lint
```

Runs all pre-commit hooks: `go-mod-tidy`, `go-test`, `go-vet`, `gofumpt`,
`golangci-lint`, and `gitleaks`. Requires `pre-commit` to be installed.

To lint a single module directly:

```bash
cd <module>
golangci-lint run --config ../.golangci.yaml --fix ./...
```

### Run all tests

```bash
# Via pre-commit (same as CI)
make lint

# Or directly across the workspace
go test -race -count=1 ./...
```

### Run tests for a single module

```bash
cd <module>
go test -race -count=1 ./...
```

### Run a single test

```bash
cd <module>
go test -race -count=1 -run TestFunctionName ./...
# Or for a sub-test:
go test -race -count=1 -run TestFunctionName/sub_test_name ./...
```

### Run an example

```bash
make example MODULE=<module>   # e.g. MODULE=config
```

### Format

```bash
# Format a module (gofumpt is stricter than gofmt — use it)
gofumpt -l -w .
```

## Code style

### Formatting

- Format with `gofumpt` (superset of `gofmt`). Running `make lint`
  applies it automatically.
- `interface{}` must be written as `any` (enforced by a gofmt rewrite
  rule in `.golangci.yaml`).
- Line length limit is **140 characters**.

### Imports

Group imports in three blocks, separated by blank lines:

```go
import (
    // 1. Standard library
    "context"
    "fmt"

    // 2. External dependencies
    "github.com/spf13/viper"

    // 3. Internal (same repo) — local-prefix set to github.com/lvlcn-t/go-kit
    "github.com/lvlcn-t/go-kit/lists"
)
```

`goimports` enforces this grouping automatically.

### Naming conventions

- Packages: short, lowercase, single word (`executors`, `lists`, `rest`).
- Exported types: `PascalCase`. Interfaces end with a role noun,
  not `-er` unless it is idiomatic (`Validator`, `Loadable`, `Effector`).
- Unexported helpers: `camelCase`. Constructors for unexported types
  are named `newFoo(...)`.
- Test files that test the same package use `package foo` (white-box).
  External-facing example tests use `package foo_test`.
- Test functions follow `TestTypeName_MethodName` or `TestFunctionName`.
  Sub-tests use `t.Run(tt.name, ...)` with a `tests []struct` table.
- Generated files (`*_moq.go`) carry a `// Code generated by moq` header
  and must not be edited manually.
- Magic numbers must be named constants; `0`, `1`, `2`, `3` are exempt
  (per `mnd` config).

### Types and generics

- Prefer generics over `any`-typed helpers when the type constraint is
  meaningful (`Filter[T any]`, `Load[T Loadable]`).
- Use `cmp.Ordered` as a constraint instead of duplicating numeric type
  switches where possible.
- Struct types that carry state should have unexported fields with
  exported accessor methods or constructors.

### Error handling

- Wrap errors with `fmt.Errorf("context: %w", err)` — never discard
  the original error.
- Define sentinel error types as structs (not `errors.New` globals) so
  callers can use `errors.Is` / `errors.As`:

  ```go
  type ErrFoo struct{ Field string }
  func (e ErrFoo) Error() string { ... }
  func (e ErrFoo) Is(target error) bool {
      _, ok := target.(ErrFoo)
      return ok
  }
  ```

- Collect multiple errors with `errors.Join`; expose the slice via
  `Unwrap() []error`.
- Use `panic` only for programming errors (e.g. unknown validation rule
  passed to a parser) — never for runtime conditions.
- Functions that cannot fail in normal usage may return the error in a
  separate named return; avoid named returns otherwise.

### Interfaces and mocks

- Keep interfaces small — one or two methods is ideal.
- Mocks are generated with [`moq`](https://github.com/matryer/moq).
  To regenerate a mock:

  ```bash
  cd <module>
  moq -out <type>_moq.go . InterfaceName
  ```

- Never hand-edit `*_moq.go` files.

### Concurrency

- Protect shared package-level variables with a `sync.Mutex` (`mu`).
  Use `defer mu.Unlock()` immediately after `mu.Lock()`.
- Functions that mutate package-level state must document whether they
  are safe for concurrent use.
- Use `golang.org/x/sync/errgroup` for fan-out goroutine coordination.

### Documentation

- Every exported symbol requires a GoDoc comment starting with the
  symbol name: `// Load loads the configuration from…`.
- Package-level doc goes on the `package` clause of one file
  (conventionally the primary file, e.g. `loader.go`, `common.go`).
- Include a runnable `// Example:` block in complex function docs.
- All content (comments, identifiers, commit messages) must be in
  English.

### Linters in use

Key enabled linters and what they enforce:

| Linter        | Enforces                                         |
| ------------- | ------------------------------------------------ |
| `errcheck`    | All errors must be checked                       |
| `gochecknoinits` | No `init()` functions                         |
| `gosec`       | Security best practices                          |
| `funlen`      | Max 50 statements per function                   |
| `gocyclo`     | Cyclomatic complexity ≤ 15                       |
| `mnd`         | No bare magic numbers (0–3 exempt)               |
| `noctx`       | HTTP requests must carry a `context.Context`     |
| `revive`      | General Go style (unused-parameter disabled)     |
| `staticcheck` | Static analysis and deprecation checks           |
| `depguard`    | `logrus` is banned; use `logutils.Log`           |

Suppress a lint warning only when unavoidable and always with a comment
explaining why (`//nolint:errcheck // intentionally ignoring…`).
